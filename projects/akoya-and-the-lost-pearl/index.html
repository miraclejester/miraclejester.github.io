About the Project

Akoya is a mermaid who transforms between land and sea forms to navigate a sunlit beach and its dark underwater caves. Built in a month for Metroidvania Month Jam 31 (theme: Magical Girls), it placed 7th out of 80 entries.

I handled all programming and level design for a four-person team, architecting the state machine, FMOD audio integration, a custom water shader, and the dual-environment movement system that makes the transformation feel mechanically distinct rather than cosmetic.

Technical Highlights

  • Node-based state machine used across the player character, all enemies, and interactables. New behaviors could be added mid-jam without touching unrelated code
  • Dual-environment movement: grounded combat mode and six-directional swim mode, each with separate physics, animation states, and hitbox behavior
  • FMOD integration via the Godot FMOD extension, with real-time crossfading between above-water and underwater soundscapes
  • Custom GLSL water shader
  • Level maps initially authored in Tiled, then migrated to Godot's native tilemap editor mid-project for better tooling in the final design stretch

Features

  • Transform into a mermaid to swim underwater
  • Fight against crabs, bats, and an evil mermaid
  • Explore a gorgeous beach and dark caves

What I Learned

Keeping state machine transitions explicit and data-driven paid off immediately when a new enemy type was needed two days before the deadline. FMOD is powerful but its Godot support was limited enough to block a web build entirely, which is worth factoring in early. Mid-jam tool migrations are a real risk: the Tiled-to-Godot switch was the right long-term call, but it should be a deliberate day-one task rather than absorbed as scope creep. The water shader was a first step into GLSL as there is more to build on there.
