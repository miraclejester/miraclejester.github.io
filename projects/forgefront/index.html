About the Project

ForgeFront blends autobattler, RTS, and roguelite mechanics. Set your factory build order, farm minerals, and push through procedurally generated maps. Between runs, minerals convert to chips that buy permanent upgrades.

Built in one week for Brackeys Jam 2026.1 by a large coordinated team of 16 credited contributors spanning directors, developers, artists, musicians, and SFX designers. Placed 613th out of 1427 entries.

I contributed as one of five developers, implementing the in-run and persistent upgrades system, doing some of the bot behaviors, fixing UI bugs, implementing bot sfx and working through the integration challenges that come with a team this size under a one-week constraint.

Technical Highlights

  • In-run upgrade systems and permanent chip-based progression between runs
  • Procedurally generated maps supporting the roguelite run structure
  • Systems-heavy architecture designed so five developers could contribute in parallel
  • Cross-functional coordination across developers, directors, artists, and SFX under a one-week deadline

Features

  • RTS and Autobattler combination
  • Roguelike runs coupled with permanent upgrades

What I Learned

Managing a large team under a one-week deadline is primarily a systems design problem: the game needs to be architected so that everyone can contribute in parallel without constant integration pain. The main lesson was that upfront system planning pays off disproportionately at this scale. Time invested on day one saves multiples later. The second half of the project was largely integration work, and the clearest takeaway is that more deliberate planning at the start would have made that phase significantly smoother. On the other hand, I was able to code a roguelike upgrade system that worked in a run-only setting as well as persistently, which was a great learning experience.
