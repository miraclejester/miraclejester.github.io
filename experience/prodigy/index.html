About the Role

Prodigy is a live-service educational MMO used in schools across the US and Canada, gamifying math and science for grades 1 to 8. Jose worked across multiple feature teams (standard features, battle system, and monetization), shipping production code to a live audience of millions.

The game runs on a custom engine built on PixiJS with TypeScript, originally written in Phaser, giving the role unusually broad exposure to both game development patterns and web engineering practices. A monetization feature Jose owned directly contributed to measurable A/B test improvements.

Key Responsibilities

  • Battle system revamp: laid the groundwork for a more engaging combat experience, reworking core architecture for extensibility
  • Tower Town: designed and implemented a focus-mode zone where players build and decorate their own town while answering more questions, deepening engagement between standard exploration loops
  • Store revamp: part of the team that made the store more data-driven, added bundle support, and built React-based internal web tooling for content management
  • Premium currency (Magicoin): contributed to the introduction of the game's premium currency system
  • IAP unification: owned the client-side aspect of unifying in-app purchase systems across web and mobile
  • UI development: built numerous interface screens across different feature areas
  • Animations: implemented character and UI animations using Spine

What I Learned

Working at Prodigy across battle, store, and monetization teams produced a developer who is comfortable with the full weight of live-service production: QA handoffs, cross-team coordination, product constraints, and shipping to a real audience where regressions have consequences. The web-native stack (PixiJS, TypeScript, React) meant game development patterns had to coexist with web engineering discipline, a combination that broadened technical range considerably.
